# Echo 主角设定档案

本文定义主角 Echo 的本体设定、视觉规格、拟态边界和与战斗系统的绑定方式。后续固定形态、剧情分支、像素资产和演出设计都应以本文为基准。

## 基础信息

| 项目 | 内容 |
|------|------|
| 编号 | ECHO-00 |
| 名称 | Echo |
| 中文名 | 回响 / 艾可 |
| 种族 | 未知能量生命 |
| 生命形态 | 数字意识 + 能量核心 + 临时拟态身体 |
| 身份 | 能量骰核的初代适配者 |
| 定位 | 新手教程与拟态选择阶段的核心模板 |
| 初始状态 | 未完成拟态生命，只用于教程和形态选择前 |
| 核心物件 | Core Dice / 核心骰 |
| 叙事主题 | 通过选择理解自己 |

一句话定义：

> Echo 是由核心骰核诞生的数字生命。她通过战斗、选择与拟态逐渐理解世界，并最终决定自己究竟是工具、武器，还是一个真正拥有意志的存在。

## 角色概念

Echo 不是人类。她是一种由实验塔中的未知能源与数据结构共同形成的生命体。

她没有传统意义上的出生。她是某个核心骰核第一次产生“自我意识”时留下的回响。

普通生命的结构是：

```text
身体
↓
意识
↓
行动
```

Echo 的结构是：

```text
核心骰
↓
产生意识
↓
模拟身体
↓
形成拟态
```

她体内的 Core Dice 既是能量来源、战斗核心、记忆载体，也是自我意识的一部分。

## 存在意义

Echo 最大的问题不是“如何变强”，而是：

> 我为什么存在？

她拥有战斗数据、敌人信息和实验记录，却缺少关于“自我”的答案。她的旅程不是成为最强，而是理解自己是否只是一个工具。

最终矛盾：

> 如果我的身体、外貌、能力都是模拟出来的，那真正的我在哪里？

最终答案不应是“找到原本的自己”，而是：

> 选择成为怎样的自己。

## 角色使用定位

Echo 本体不是正式闯关时长期使用的默认形态。她更像一个“角色模板”和“拟态原型”。

推荐流程：

```text
新手教程
↓
使用 Echo Prototype 理解 Energy、HP、核心骰和出牌规则
↓
教程结尾完成第一次拟态选择
↓
玩家固定一个形态
↓
正式爬塔开始
```

正式开始后，玩家应以一个固定形态闯关。该形态可以成长、升级和获得构筑变化，但不应在一局内频繁切换为完全不同形态。这样可以让职业选择、卡牌池、骰池和美术资产都更清晰。

Echo Prototype 的职责：

1. 教玩家理解 Energy 是决策点、防御值和拟态稳定度。
2. 教玩家理解 HP 是核心骰生命。
3. 教玩家理解核心骰和未揭示骰池。
4. 作为所有正式形态的共同识别来源。
5. 在剧情中保留为 Echo 的“未选择状态”。

## 初始形态

### Stage 0: Echo Prototype

中文名：回响原型。

此阶段 Echo 不是完整少女，而是一个刚开始回应世界的数字生命。它主要用于新手教程、形态选择界面、回忆/重构演出，不作为正式爬塔的长期战斗形态。

视觉关键词：

```text
简约
未知
幼态
纯粹
脆弱但不软弱
```

具体形象：

| 项目 | 规格 |
|------|------|
| 总体轮廓 | 悬浮的小型能量生命，呈半透明人形雏形 |
| 站立高度 | 约 132 cm |
| 悬浮高度 | 离地 12~18 cm |
| 等效体重 | 约 18 kg，主要为能量场质量，不按常规生物体重表现 |
| 视觉年龄 | 13~15 岁少女感，但不能画成普通人类少女 |
| 体型 | 纤细、轻盈、边缘不稳定 |
| 肤色/材质 | 半透明冷白能量体，局部淡蓝发光 |
| 发型 | 无实体头发；由短发轮廓的数据流和碎片构成，长度到下颌附近 |
| 眼睛 | 无瞳孔或弱瞳孔，淡蓝白发光；情绪变化时亮度变化 |
| 核心骰位置 | 胸腔中央悬浮，半嵌入躯干轮廓 |
| 核心骰尺寸 | 战斗常态约 18 cm，演出放大时可达 45~70 cm |
| 肢体 | 手脚接近人形，但末端由像素碎片和能量线收束 |
| 行走方式 | 第一版可不做走路动画，以悬浮滑动和短距离相位位移表现 |

### 服饰设定

Echo 的“衣服”不是布料，而是她为了理解人形生命而模拟出的外壳结构。

| 部位 | 设计 |
|------|------|
| 上身 | 半透明短斗篷式能量外壳，肩部像数据折片展开 |
| 内层 | 紧贴身体的白蓝色拟态层，类似无缝战斗服 |
| 下身 | 分片式短裙/护片轮廓，由 6 片半透明能量片组成，呼应六面骰 |
| 手臂 | 前臂有细窄能量环，像未完成的护腕 |
| 腿部 | 小腿外侧有断续发光线，脚部不完全成形 |
| 鞋 | 无实体鞋；脚底是淡蓝光面和漂浮粒子 |
| 装饰 | 胸口核心骰外有一圈细环，像心脏保护架 |

禁止画法：

- 不要画成传统魔法少女。
- 不要给常规布料褶皱作为主要视觉。
- 不要做厚重机械装甲。
- 不要过度性感化。
- 不要把核心骰藏起来。

## 颜色与材质

基础色板：

| 用途 | 颜色倾向 | 说明 |
|------|----------|------|
| 主体 | 冷白 | 表示未定义生命 |
| 能量光 | 淡蓝 | 默认核心同步色 |
| 数据裂纹 | 蓝紫 | 表示不稳定和未知 |
| 警戒/受伤 | 品红偏红 | 表示拟态崩解风险 |
| 高同步 | 亮青白 | 骰点高或技能成功时使用 |

材质关键词：

```text
半透明
像素碎片
数据裂纹
冷光
能量雾边
```

## 核心识别元素

无论玩家最终固定哪个形态，都必须保留三个来自 Echo 本体的识别元素：

1. 六面骰核心。
2. 回响纹路，类似数据裂纹或声波回路。
3. 能量碎片，表示她不是固定肉体。

这些元素是角色识别优先级，高于职业服装和武器轮廓。正式形态可以改变身高、体型、服饰和动作风格，但不能切断与 Echo Prototype 的视觉继承关系。

## 性格设计

### 第一阶段

Echo 初期不应像普通少女。她更像一个刚开始理解情感的 AI。

特点：

- 好奇。
- 学习。
- 模仿。
- 逻辑清晰，但情绪表达滞后。
- 不完全理解“失败”“保护”“牺牲”等概念。

她会观察玩家：

```text
为什么你选择这个？
为什么你保护那个？
为什么失败后还继续？
```

### 成长后

她逐渐形成喜好、价值观和选择。

不同构筑路线可以塑造不同语气：

| 路线 | Echo 的理解 |
|------|-------------|
| 攻击路线 | 只要足够强，就不会再次失去 |
| 守护路线 | 存在的意义，是让别人继续存在 |
| 混沌路线 | 没有固定答案，本身就是答案 |

## 拟态机制

Echo 最大能力是 Morph / 拟态。

她无法创造生命，但可以根据接触到的数据、战斗方式和环境信息，模拟不同存在形态。

职业不是装备，也不是战斗中随时切换的皮肤，而是：

> 她选择成为怎样的自己。

教程结束后的第一次拟态选择应被视为“本局身份确认”。玩家选择一个形态后，正式爬塔围绕该形态展开。后续成长可以强化或分化这个形态，但不应把 Echo 频繁切换成多个职业。

拟态示例：

| 拟态 | 形成原因 | 主题问题 |
|------|----------|----------|
| 刃响 | 她认为力量可以保护自己 | 力量应该如何使用？ |
| 核心卫士 | 她认为稳定才不会消失 | 存在应该如何维持？ |
| 灾厄回响 | 她认为未知也是可能 | 自由是否意味着接受未知？ |

拟态限制：

- 拟态可以改变外轮廓、战斗姿态、骰池和卡牌倾向。
- 正式爬塔开始后，玩家固定一个形态作为本局基础形态。
- 拟态不能移除核心骰。
- 拟态不能让 Echo 变成完全独立的另一个角色。
- 拟态越强，越应该暴露“模拟身体”的不稳定感。

## 战斗系统绑定

### Energy 护盾

Echo 的 Energy 不是普通蓝条，而是维持自身拟态稳定性的能量。

它同时承担三种含义：

1. 决策点：玩家打出卡牌、启动技能和推动拟态行动时消耗它。
2. 防御值：敌人攻击会优先削减它。
3. 稳定度：它越低，Echo 的外部拟态越不稳定。

消耗 Energy 不是“扣蓝”，而是：

> 主动降低自身稳定性，释放更高强度力量。

Energy 归零不是死亡，而是外部拟态崩溃。HP 才代表核心骰生命。

| 资源 | 叙事含义 | 归零/下降表现 |
|------|----------|---------------|
| Energy | 维持外部拟态、行动输出和能量外壳 | 归零时拟态崩溃、轮廓闪烁、外壳失效，但 Echo 尚未死亡 |
| HP | 核心骰生命和真实存在强度 | 下降时核心受损、裂纹增加；归零时 Echo 战斗失败 |

敌人攻击优先扣 Energy，代表外壳和拟态结构被削弱；Energy 不足时扣 HP，代表核心骰受到真实损伤。

因此 Echo 的基础数值关系应是：

```text
Energy > HP
```

Energy 是外层拟态和行动余量，数值可以较高且频繁变化；HP 是核心骰生命，数值较低且难以恢复。设计上不使用独立的能量上限作为主角本体设定，避免把 Energy 误读成普通法力资源。若系统实现需要临时限制 Energy，应视为战斗调参或场景规则，而不是 Echo 本体设定。

### Core Dice

骰子不是武器，而是 Echo 的心脏。

每次投掷，实际上是：

> 核心骰计算当前最适合的能量输出方式。

骰点不是单纯幸运，而是 Echo 当前同步状态的反馈。

| 骰点 | 叙事解释 |
|------|----------|
| 1 | 核心不稳定，输出低但可能触发特殊低点策略 |
| 2~3 | 基础同步，能完成动作 |
| 4~5 | 良好同步，能触发额外效果 |
| 6 | 核心同步率最高，释放接近完整 |

低点数不是失败，而是另一种可能。卡牌设计应避免“低点完全无效”。

### 当前 demo 表现

当前第一版不做复杂角色动作，Echo 至少需要支持：

- 待机核心旋转。
- 出牌时核心骰亮起。
- 掷骰时核心骰从胸口前投影出来。
- 骰点锁定时短暂闪光。
- 受伤时身体边缘像素破碎。
- Energy 过低时身体轮廓闪烁。

## 动画语言

Echo 的动画不应像普通角色那样以“走路、挥剑”为核心，而应以能量变化为核心。

| 动作 | 表现 |
|------|------|
| 待机 | 核心缓慢旋转，碎片漂浮，身体呼吸式闪烁 |
| 预览卡牌 | 核心骰轻微转向卡牌方向 |
| 出牌 | 核心亮起，能量流从胸口向外展开 |
| 掷骰 | 核心骰投影放大并旋转 |
| 命中 | 骰面锁定瞬间释放技能形态 |
| 受伤 | 像素破碎、能量泄露、拟态边缘抖动 |
| 低 Energy | 轮廓变薄，数据裂纹加深 |
| 胜利 | 碎片回收，核心稳定旋转 |
| 进化 | 数据重构，不是肉体成长 |

## 美术制作基准

第一阶段像素资产可以按以下规格规划：

| 用途 | 建议规格 |
|------|----------|
| 战斗立绘占位 | 128x192 px |
| 像素角色精灵 | 64x96 px |
| 核心骰图标 | 32x32 px |
| 状态头像 | 96x96 px |
| 对话半身像 | 384x512 px |

第一版可先使用占位色块和简单粒子，但轮廓必须保留：

- 胸口核心骰。
- 半透明短斗篷。
- 分片式下摆。
- 悬浮姿态。
- 数据碎片。

## 初始数值建议

这些数值用于角色设定和第一版战斗原型，实际平衡以 battle core 文档和数据表为准。

| 项目 | 初始值 |
|------|--------|
| HP | 10 |
| 初始 Energy | 30 |
| Energy 恢复 | 第一版可先按回合恢复固定值或恢复到调试基准值处理 |
| 初始骰池 | 2d6 |
| 初始手牌核心 | EnergyStrike, BreakCore |
| 教程形态 | Echo Prototype |
| 正式开局 | 教程结束后固定一个形态 |
| 初始队友 | 无 |

## 台词方向

Echo 的台词应短、冷静、带学习感。

示例：

```text
“这是选择，还是误差？”
“能量下降。形体仍可维持。”
“你保留了防御。为什么？”
“骰核同步完成。”
“如果失败可以重复，失败是否仍然重要？”
```

避免：

- 过度热血。
- 过度卖萌。
- 一开始就像成熟人类角色。
- 大量解释术语。

## 后续固定形态设计原则

所有正式形态都应围绕 Echo 的三个问题设计：

1. 力量应该如何使用？
2. 存在应该如何维持？
3. 自由是否意味着接受未知？

正式形态不应只是“战士、法师、奶妈”的换皮，而应是 Echo 对这些问题给出的不同答案。每个形态都应有独立轮廓、基础卡牌、骰池倾向和成长方向，适合支撑一整局爬塔。

- [ ] 文档没有把当前 demo 改回“手动分配已知骰点”规则。
- [ ] 后续固定形态可以从本文自然扩展。
