# Card Subtype System Design

## 概述

本文档定义游戏的卡牌子类型系统，将卡牌分为四大类别：基础卡牌、技能卡牌、消耗品卡牌和其他卡牌。每个类别具有独特的机制属性、游戏功能和视觉标识。

---

## 一、基础卡牌 (Basic Cards)

### 1.1 类别定义

基础卡牌是战斗中最直接的交互手段，主要用于攻击和防御。它们是玩家卡组的核心组成部分，通常具有较低的资源消耗和明确的效果。

### 1.2 攻击卡牌 (Attack Cards)

#### 游戏目的与策略角色
- 核心输出手段，直接造成伤害
- 配合骰子系统实现可变伤害
- 可附带状态效果或特殊触发条件

#### 统计参数与效果值

| 参数名称 | 类型 | 说明 | 默认值范围 |
|---------|------|------|-----------|
| BaseDamage | int | 基础伤害值 | 1-10 |
| DamageFormula | string | 伤害计算公式 | "dice + X", "dice * X" |
| DiceCost | int | 骰子消耗 | 0-2 |
| EnergyCost | int | 能量消耗 | 0-3 |
| EffectCondition | string | 触发条件 | "dice >= X", "dice_sum >= X" |
| EffectDescription | string | 附加效果描述 | - |

#### 视觉设计规范
- 卡牌边框颜色：红色 (#FF4444)
- 卡牌图标：剑、斧、拳等攻击类图标
- 卡牌背景：深色渐变，带有攻击特效纹理
- 伤害数值显示：醒目的红色字体

#### 交互规则
- 攻击卡牌只能对敌方目标使用
- 伤害计算优先应用破甲状态加成
- 附加效果在伤害结算后触发

#### 获取方式与卡组限制
- 初始卡组包含
- 战斗奖励获取
- 酒馆购买
- 卡组中攻击卡数量上限：无硬性限制，但建议不超过总卡牌数的60%

#### 平衡考虑
- 低消耗高伤害卡牌应有随机性（依赖骰子）
- 固定高伤害卡牌应有较高消耗
- 攻击卡平均效率比（伤害/消耗）应控制在1.5-2.5之间

### 1.3 防御/闪避卡牌 (Defense/Dodge Cards)

#### 游戏目的与策略角色
- 保护玩家免受伤害
- 创造反击机会
- 管理能量资源（Energy作为护盾）

#### 统计参数与效果值

| 参数名称 | 类型 | 说明 | 默认值范围 |
|---------|------|------|-----------|
| ShieldValue | int | 护盾值 | 2-10 |
| EvasionRate | float | 闪避率 | 0.2-0.5 |
| CounterDamage | int | 反击伤害 | 0-5 |
| DiceCost | int | 骰子消耗 | 0-1 |
| EnergyCost | int | 能量消耗 | 0-2 |
| Duration | int | 持续回合数 | 1-3 |

#### 视觉设计规范
- 卡牌边框颜色：蓝色 (#4444FF)
- 卡牌图标：盾牌、闪避箭头、防御姿态
- 卡牌背景：明亮渐变，带有护盾特效纹理
- 护盾数值显示：蓝色字体，带有盾牌图标

#### 交互规则
- 防御卡牌只能对玩家自身使用
- 护盾优先于Energy吸收伤害
- 闪避判定在伤害计算前进行
- 反击伤害在成功闪避后触发

#### 获取方式与卡组限制
- 战斗奖励获取
- 酒馆购买
- 事件奖励
- 卡组中防御卡数量上限：无硬性限制，但建议不超过总卡牌数的40%

#### 平衡考虑
- 高护盾值卡牌应有较高消耗
- 闪避率不应超过50%
- 防御卡平均效率比（护盾/消耗）应控制在2.0-3.0之间

## 二、技能卡牌 (Skill Cards)

### 2.1 类别定义

技能卡牌用于施加状态效果、改变战斗局势。它们不直接造成大量伤害，但能通过增益/减益影响整个战斗进程。

### 2.2 正面增益卡牌 (Positive Buff Cards)

#### 游戏目的与策略角色
- 增强玩家属性
- 提供持续战斗优势
- 配合攻击卡牌发挥最大效果

#### 统计参数与效果值

| 参数名称 | 类型 | 说明 | 默认值范围 |
|---------|------|------|-----------|
| BuffType | enum | 增益类型 | AttackUp, DefenseUp, EnergyRegen, DiceBonus |
| BuffAmount | int | 增益数值 | 1-5 |
| Duration | int | 持续回合数 | 2-5 |
| StackLimit | int | 堆叠上限 | 1-3 |
| StackBehavior | enum | 堆叠行为 | RefreshDuration, AddStacks, Replace |
| DiceCost | int | 骰子消耗 | 0-1 |
| EnergyCost | int | 能量消耗 | 1-3 |

#### 视觉设计规范
- 卡牌边框颜色：绿色 (#44FF44)
- 卡牌图标：向上箭头、光环、加号
- 卡牌背景：温暖渐变，带有光芒特效
- 增益数值显示：绿色字体，带有加号图标

#### 交互规则
- 正面增益卡牌只能对玩家自身使用
- 增益效果在回合开始时生效
- 堆叠规则：同类型增益可堆叠，但受限于StackLimit
- 持续时间在玩家回合结束时减少

#### 获取方式与卡组限制
- 战斗奖励获取
- 酒馆购买
- 队友提供
- 卡组中增益卡数量上限：建议不超过总卡牌数的30%

#### 平衡考虑
- 高数值增益应有较短持续时间
- 持续时间越长，消耗越高
- 增益效果不应超过基础属性的50%

### 2.3 负面减益卡牌 (Negative Buff Cards)

#### 游戏目的与策略角色
- 削弱敌人能力
- 创造输出窗口
- 控制战斗节奏

#### 统计参数与效果值

| 参数名称 | 类型 | 说明 | 默认值范围 |
|---------|------|------|-----------|
| DebuffType | enum | 减益类型 | Vulnerable, Weak, Slow, ArmorBreak |
| DebuffAmount | int | 减益数值 | 1-5 |
| Duration | int | 持续回合数 | 2-4 |
| ResistChance | float | 抗性几率 | 0.0-0.3 |
| DiceCost | int | 骰子消耗 | 0-1 |
| EnergyCost | int | 能量消耗 | 1-3 |

#### 视觉设计规范
- 卡牌边框颜色：紫色 (#AA44FF)
- 卡牌图标：向下箭头、骷髅、锁链
- 卡牌背景：暗紫色渐变，带有腐蚀特效
- 减益数值显示：紫色字体，带有减号图标

#### 交互规则
- 负面减益卡牌只能对敌方目标使用
- 减益效果在敌人回合开始时生效
- 敌人可能拥有抗性，有几率抵抗减益
- 持续时间在敌人回合结束时减少

#### 获取方式与卡组限制
- 战斗奖励获取
- 酒馆购买
- 事件奖励
- 卡组中减益卡数量上限：建议不超过总卡牌数的30%

#### 平衡考虑
- 强力减益应有较高消耗或触发条件
- 抗性机制防止减益效果过于强大
- 减益效果不应完全削弱敌人能力

## 三、消耗品卡牌 (Consumable Cards)

### 3.1 类别定义

消耗品卡牌是一次性使用的资源卡，使用后从卡组中移除。它们提供即时的资源恢复或特殊效果。

### 3.2 游戏级消耗品 (Game-Level Consumables)

#### 游戏目的与策略角色
- 整个游戏会话中有限使用
- 关键时刻的救命稻草
- 战略性资源管理

#### 统计参数与效果值

| 参数名称 | 类型 | 说明 | 默认值范围 |
|---------|------|------|-----------|
| MaxUsage | int | 游戏内最大使用次数 | 1-3 |
| CurrentUsage | int | 当前已使用次数 | 0 |
| EffectType | enum | 效果类型 | FullHeal, FullEnergy, RemoveAllDebuffs |
| EffectValue | int | 效果数值 | - |
| EnergyCost | int | 能量消耗 | 0 |

#### 视觉设计规范
- 卡牌边框颜色：金色 (#FFD700)
- 卡牌图标：宝箱、药瓶、宝石
- 卡牌背景：金色渐变，带有闪光特效
- 使用次数显示：右上角金色数字

#### 交互规则
- 使用后从手牌移除，不进入弃牌堆
- 游戏级消耗品全局共享使用次数
- 不同游戏级消耗品独立计数
- 在战斗外也可使用

#### 获取方式与卡组限制
- 游戏开始时给予
- 事件奖励
- 商店购买（稀有）
- 卡组中游戏级消耗品数量上限：3张

#### 平衡考虑
- 强力效果限制使用次数
- 获取难度较高
- 使用决策需要谨慎权衡

### 3.3 战斗级消耗品 (Battle-Level Consumables)

#### 游戏目的与策略角色
- 每场战斗中有限使用
- 提供即时资源恢复
- 战斗内策略调整

#### 统计参数与效果值

| 参数名称 | 类型 | 说明 | 默认值范围 |
|---------|------|------|-----------|
| UsesPerBattle | int | 每场战斗使用次数 | 1-2 |
| EffectType | enum | 效果类型 | Heal, EnergyRestore, DrawCards, GainDice |
| EffectValue | int | 效果数值 | 5-15 |
| EnergyCost | int | 能量消耗 | 0-1 |

#### 视觉设计规范
- 卡牌边框颜色：橙色 (#FF8800)
- 卡牌图标：卷轴、药水、能量球
- 卡牌背景：橙色渐变，带有脉动特效
- 使用次数显示：右上角橙色数字

#### 交互规则
- 使用后从手牌移除，不进入弃牌堆
- 战斗级消耗品每场战斗重置使用次数
- 不同战斗级消耗品独立计数
- 只能在战斗中使用

#### 获取方式与卡组限制
- 战斗奖励获取
- 酒馆购买
- 事件奖励
- 卡组中战斗级消耗品数量上限：5张

#### 平衡考虑
- 效果数值适中，不会破坏战斗平衡
- 每场战斗有限制，防止过度依赖
- 消耗品策略是资源管理的重要部分

## 四、其他卡牌 (Other Cards)

### 4.1 装备卡牌 (Equipment Cards)

#### 游戏目的与策略角色
- 提供永久性或半永久性属性提升
- 改变玩家战斗风格
- 装备槽限制增加策略深度

#### 统计参数与效果值

| 参数名称 | 类型 | 说明 | 默认值范围 |
|---------|------|------|-----------|
| EquipmentSlot | enum | 装备槽位 | Weapon, Armor, Accessory |
| EffectType | enum | 效果类型 | PermanentBuff, PassiveEffect, TriggeredEffect |
| EffectValue | int | 效果数值 | 1-5 |
| IsPermanent | bool | 是否永久 | true/false |
| Duration | int | 持续战斗数（非永久） | 3-5 |
| EnergyCost | int | 装备能量消耗 | 2-4 |

#### 视觉设计规范
- 卡牌边框颜色：银色 (#CCCCCC)
- 卡牌图标：武器、护甲、饰品
- 卡牌背景：金属质感渐变，带有光泽特效
- 装备槽位显示：底部标签

#### 交互规则
- 打出后装备到对应槽位，从手牌移除
- 每个装备槽位只能装备一张卡牌
- 装备效果立即生效
- 新装备会替换同槽位旧装备

#### 获取方式与卡组限制
- 战斗奖励获取（稀有）
- 商店购买
- 事件奖励
- 卡组中装备卡数量上限：3张（各槽位1张）

#### 平衡考虑
- 强力装备应有较高消耗
- 装备槽限制防止过度强化
- 永久装备获取难度较高

### 4.2 诅咒卡牌 (Curse Cards)

#### 游戏目的与策略角色
- 提供负面效果，增加游戏难度
- 强制玩家调整策略
- 影响卡组构成和抽牌概率

#### 统计参数与效果值

| 参数名称 | 类型 | 说明 | 默认值范围 |
|---------|------|------|-----------|
| CurseType | enum | 诅咒类型 | HandSizeReduction, DrawPenalty, EnergyDrain |
| EffectValue | int | 效果数值 | 1-3 |
| Duration | int | 持续战斗数 | 2-5 |
| RemovalCondition | string | 移除条件 | "Discard X cards", "Take X damage" |
| IsRemovedOnDiscard | bool | 弃牌时移除 | true/false |

#### 视觉设计规范
- 卡牌边框颜色：黑色 (#222222)
- 卡牌图标：骷髅、幽灵、黑暗符文
- 卡牌背景：深色渐变，带有腐蚀特效
- 诅咒标识：卡牌角落显示诅咒图标

#### 交互规则
- 诅咒卡牌加入卡组后，玩家无法主动打出
- 抽到时占用手牌位置，但无法使用
- 只能通过移除条件或战斗结束移除
- 某些诅咒在抽到或弃牌时触发效果

#### 获取方式与卡组限制
- 负面事件获得
- 战斗失败惩罚
- 某些奖励选项
- 卡组中诅咒卡数量上限：无硬性限制，但建议不超过总卡牌数的15%

#### 平衡考虑
- 诅咒效果不应过于强大，避免无法翻盘
- 移除条件应合理可行
- 诅咒数量控制在可管理范围内

## 六、数值平衡框架

### 6.1 效率计算公式

```text
效率比 = 效果数值 / (EnergyCost + DiceCost * 1.5)
```

### 6.2 各类别效率基准

| 类别 | 效率比基准 | 说明 |
|------|-----------|------|
| Attack Cards | 1.5-2.5 | 伤害/消耗 |
| Defense Cards | 2.0-3.0 | 护盾/消耗 |
| Buff Cards | 1.0-1.5 | 增益数值/消耗/持续时间 |
| Debuff Cards | 1.0-1.5 | 减益数值/消耗/持续时间 |
| Consumables | 3.0-5.0 | 效果/使用次数限制 |
| Equipment | 2.0-3.0 | 属性提升/消耗/装备槽限制 |

### 6.3 骰子影响修正

```text
实际效率 = 基础效率 * (1 + 骰子不确定性系数)
骰子不确定性系数 = (MaxDamage - MinDamage) / MaxDamage * 0.3
```
